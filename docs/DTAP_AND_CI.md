# DTAP and CI Evidence

This project uses a GitHub Actions development lane in:
- `.github/workflows/ci.yml`

## Development (D)
- Developers work in feature branches and run local compile/tests.
- CI triggers on `push` to `develop` and on pull requests to `main`/`develop`.

## Testing (T)
Automated in CI:
- Java compile for source and tests.
- Unit test execution via `AllTests`.
- Code coverage report generation (JaCoCo: HTML/XML/CSV).
- Coverage percentage summary output (`coverage-summary.txt`).
- Static quality report (`javac -Xlint`) exported as artifact.

## Acceptance (A)
- Pull request workflow to `main`/`develop` acts as acceptance gate with required automated checks.

## Production (P)
- CI builds a runnable JAR (`release/JabberPoint.jar`) and uploads it as an artifact.
- Full deployment to a live production environment is still a follow-up step.

## CI Artifacts
Generated by workflow:
- `coverage-html/`
- `coverage.xml`
- `coverage.csv`
- `coverage-summary.txt`
- `lint-report.txt`
- `release/JabberPoint.jar`
